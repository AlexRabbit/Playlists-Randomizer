## рҹӨ” What is this?

**Playlists Randomizer** is a free web app that lets you:

1. **Paste YouTube playlist links** (one or many)
2. **Press play** and hear videos in **random order** (or sequential)
3. **Organize** everything into **Lists** and **Cards** вҖ” like folders and players
4. **Save everything** by **bookmarking the page** (no login needed)

> **Think of it like:** Spotify shuffle, but for *your* YouTube playlists вҖ” and you control the layout.

**You do NOT need to:**
- Create an account
- Install anything (if you use the live website)
- Pay for hosting
- Trust a server with your data вҖ” it all stays in **your browser**

## вҡЎ Try it in 30 seconds (zero install)

1. Open the live app: **[alexrabbit.github.io/Playlists-Randomizer](https://alexrabbit.github.io/Playlists-Randomizer/)**
2. On the **right**, type a list name (e.g. `Test`) вҶ’ click **`+`**
3. In the **center**, type a card name вҶ’ click **`+ Add Card`**
4. Click **`Edit playlists`** on the card
5. Paste a **public** YouTube playlist URL, for example:
   ```
   https://www.youtube.com/playlist?list=PLrAXtmRdnEQy6nuLMHjMZ6s3fKjTjJ4T
   ```
6. Click **`Save & Play`**
7. Press **в–¶ Play**

рҹҺү You are now listening to a shuffled playlist.

## рҹ“– Complete beginner tutorial (step by step)

<details open>
<summary><strong>Step 1 вҖ” Open the app</strong></summary>

<br>

- **Easiest:** use the [live GitHub Pages link](https://alexrabbit.github.io/Playlists-Randomizer/)
- **On your computer (developers):** see [Run locally](#-for-developers-run-on-your-computer) below

You need a modern browser: **Chrome**, **Firefox**, **Edge**, or **Safari**.

</details>

<details>
<summary><strong>Step 2 вҖ” Create your first List (right sidebar)</strong></summary>

<br>

1. Look at the **right panel** вҖ” header says **LISTS**
2. Type a name in the box (e.g. `Relax`, `Gym`, `Study`)
3. Click the red **`+`** button
4. Your list appears below вҖ” click it to select it

**Tips:**
- **Double-click** a list name to rename it
- Drag the **в ҝ** handle to reorder lists
- **Right-click** a list вҶ’ plays all cards in that list

</details>

<details>
<summary><strong>Step 3 вҖ” Add a Card (center area)</strong></summary>

<br>

1. With a list selected, find **"+ Add Card"** at the top of the center
2. Type a card name (e.g. `Ocean sounds`, `80s hits`)
3. Click **Add Card**

A **card** is one player. Each card can load **multiple YouTube playlists** and merge them into one shuffle queue.

</details>

<details>
<summary><strong>Step 4 вҖ” Add YouTube playlists to the card</strong></summary>

<br>

1. On the card, open the **вӢ®** menu вҶ’ **Edit playlists**  
   *(or use the edit control on the card)*
2. In the text box, paste playlist URLs or IDs вҖ” **one per line**

**All of these work:**

```
https://www.youtube.com/playlist?list=PLxxxxxxxx
https://youtube.com/playlist?list=PLxxxxxxxx
PLxxxxxxxx
```

3. Click **Save & Play**

The app fetches video titles and builds your queue. The first video loads automatically.

**Bulk paste:** use **Bulk paste playlists** to dump many URLs at once (comma, semicolon, or newline separated).

</details>

<details>
<summary><strong>Step 5 вҖ” Use the player controls</strong></summary>

<br>

Each card has two rows of controls:

| Button | What it does |
|--------|----------------|
| **вҸ® Prev** | Previous video in queue |
| **в–¶ / вҸё** | Play or pause |
| **вҸӯ Next** | Next video |
| **рҹ”Ҡ Volume** | Volume slider |
| **Random / Sequential** | Shuffle order vs playlist order |
| **Show video** | Toggle video on/off (audio-only mode) |
| **Autoplay next** | Auto-advance when a video ends |
| **рҹ”Қ Search** | Find a video by title in this card's queue |

**Seek bar:** drag to jump inside the current video.

</details>

<details>
<summary><strong>Step 6 вҖ” Open the playlist sidebar (left panel)</strong></summary>

<br>

1. Turn **Show video** **ON** on a card
2. The **left panel** slides in вҖ” same glass style as the right sidebar
3. You see:
   - **Now playing** вҖ” pinned at top (stays visible while you scroll)
   - **Full queue** вҖ” click any row to jump to that video
   - **в ҝ Drag handle** вҖ” drag rows to reorder the queue
   - **Badges** вҖ” `Deleted`, `Private`, `Unavailable` on broken videos (auto-skipped on play)
   - **Load full playlist** вҖ” footer button if YouTube only returned part of the list

Turn **Show video** **OFF** вҶ’ left panel closes.

</details>

<details>
<summary><strong>Step 7 вҖ” Save your work (IMPORTANT)</strong></summary>

<br>

**Everything lives in the browser URL.** No cloud account.

### Option A вҖ” Bookmark (recommended)

1. Right sidebar вҶ’ **Copy bookmark URL** (or drag **вҳ… Drag to Bookmarks**)
2. Save as a browser bookmark
3. Next time: open that bookmark вҶ’ your lists, cards, and settings return

### Option B вҖ” Export backup file

1. Right sidebar вҶ’ **Export backup**
2. A `.json` file downloads to your computer
3. **Import backup** restores it later (or on another device)

> вҡ пёҸ If you close the tab **without** bookmarking or exporting, unsaved work may be lost.

</details>

<details>
<summary><strong>Step 8 вҖ” Search across a whole list</strong></summary>

<br>

1. Select a list
2. Click **рҹ”Қ Search** in the center header
3. Type **3+ characters** of a video title
4. Click a result вҶ’ the focused card jumps to that video

</details>

<details>
<summary><strong>Step 9 вҖ” Optional: YouTube API key (full playlists)</strong></summary>

<br>

Without an API key, YouTube limits how many videos load per playlist (~**200** via Innertube, sometimes less).

**To load complete playlists:**

1. [Google Cloud Console](https://console.cloud.google.com/) вҶ’ enable **YouTube Data API v3** вҶ’ create an API key
2. In the app: right sidebar вҶ’ **Settings** вҶ’ paste key вҶ’ **Save**
3. The key is stored in **your bookmark URL** (client-side only)

Or click **Load full playlist** on a card / in the left sidebar after a truncated load.

</details>

## рҹ”’ Privacy & data

|----------|--------|

## вқ“ FAQ & troubleshooting

<details>
<summary><strong>Nothing plays / "No videos"</strong></summary>

<br>

- Playlist must be **public** (not private/unlisted-only)
- Check URLs are correct `playlist?list=PL...` format
- Try **Load full playlist** or add a **YouTube API key**
- Open **рҹ”¬ Logs** (top bar) for error details

</details>

<details>
<summary><strong>Only ~15вҖ“200 videos loaded</strong></summary>

<br>

YouTube limits anonymous fetches. Fixes:

1. **Settings** вҶ’ paste **YouTube Data API v3** key вҶ’ Save  
2. Or click **Load full playlist** after load

</details>

<details>
<summary><strong>My bookmark URL is huge</strong></summary>

<br>

Normal for big workspaces. The app uses compression (`lz-string`). Very large workspaces move data to `#ws=` in the hash to keep the link manageable.

</details>

<details>
<summary><strong>Video says Deleted / Private</strong></summary>

<br>

The app labels it, auto-skips on playback, and continues to the next playable video.

</details>

<details>
<summary><strong>Blank page after deploy</strong></summary>

<br>

`VITE_BASE_PATH` must match your repo name, e.g. `/Playlists-Randomizer/`. See [`docs/DEPLOY.md`](docs/DEPLOY.md).

</details>

<details>
<summary><strong>Can I use this on phone?</strong></summary>

<br>

Yes вҖ” the layout adapts. Sidebars stack on narrow screens. For best experience, bookmark on mobile Safari/Chrome.

</details>
